组件的生命周期

组件(component)是React的核心,必须调用React.createClass来创建.
一个组件就是一个状态机,它有三种不同的状态:初始化(实例化)阶段,运行阶段,销毁阶段,React为每个阶段都提供了不同的钩子(hook)函数来作为响应.

初始化阶段

一个组件初次被创建时,会依次调用下面的五个方法:

  • getDefaultProps
  • getInitialState
  • componentWillMount
  • render
  • componentDidMount

getDefaultProps
该方法是用来获取实例初始化之前的属性,它只会被调用一次,且会在所有的实例中共享.对于那些没有被父组件指定props的新建实例来说,这个方法返回的对象可用于为实例设置默认的props值.
getInitialState
该方法用来初始化每个实例特有的状态state.对于组件的每个实例来说,它的调用次数有且仅有一次.
componentWillMount
该方法会在完成首次渲染之前被调用,并且这是再render方法调用前最后一次修改组件状态的机会.
render
组件唯一一个必须的方法.它将会创建一个虚拟的DOM,用来表示组件的输出.该方法需满足下面几点:

  • 只能通过this.props和this.state来访问数据;
  • 只能有一个顶层组件(不能返回一组元素);
  • 不允许修改组件的状态和DOM输出.
    componentDidMount
    在真实的DOM已经被成功渲染后,可以在该方法内部通过this.getDOMNode来访问到它.

运行阶段

组件已经被渲染好,并且用户可以与它交互了.通常是通过一次事件来触发一个事件处理器,随着用户改变了组件或是整个应用的状态,便有新的state流入组件树,从而有机会去操控它.

  • componentWillReceiveProps
  • shouldComponentUpdate
  • componentWillUpdate
  • render
  • componentDidUpdate

componentWillReceiveProps
在任意时刻,组件的props都可以通过父组件来修改,在此之前,该方法会被调用,可以来更改props对象及更新state.
shouldComponentUpdate
如果确切的知道某个组件或者它的任何子组件不需要渲染新的props或state,将该方法返回false,则可以阻止render方法,以及位于render前后的钩子函数(该函数是非必需的,大多情况下没必要在开发中使用).
componentWillUpdate
和componentWillMount方法类似,组件会在接收到新的props或者state进行渲染之前调用该方法.
注:不可在该方法中更新属性或状态,应该借助componentWillReceiveProps在运行时更新state.
render
同初始化阶段
componentDidUpdate
和componentDidMount类似,该方法可以更新已经渲染好的DOM.

销毁阶段

销毁阶段只有一个钩子函数,即componentWillUnMount.
每当React使用完一个组件,这个组件就必须从DOM中卸载随后销毁.在销毁前,该方法会被触发.我们可以在其中做一些清理工作,在componentDidMount中添加的所有任务都需要在该方法中撤销,比如创建的计时器或事件监听器等.
